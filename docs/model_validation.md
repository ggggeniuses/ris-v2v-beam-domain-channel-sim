# Model Validation

## RIS-V2V Correlation

The spatial CCF, temporal ACF, and frequency FCF baselines are generated by the
project functions under `src/ccf`, `src/acf`, and `src/fcf`. Tests require all
correlation outputs to be finite and bounded in `[0, 1]`.

## Capacity

The array-domain channel and its DFT beam-domain representation have the same
ideal MIMO capacity up to numerical precision because the transform is unitary.
The capacity figure is therefore an invariance check, not a capacity-gain claim.

## Sparsity and Complexity

Beam-domain concentration is quantified by top-K energy, magnitude thresholds,
and the number of coefficients required for 90%, 95%, and 99% energy capture.

Full GBSM and full BDCM complexity totals use the same `P*Q` coefficient count.
The sparse-BDCM curve uses the measured coefficient count required for 95%
energy capture. This keeps the operation-count comparison on a consistent
basis.

## FAS-UAV

The modeling-error experiment evaluates raw FAS and ULA complex CIRs. Capacity
experiments use Frobenius-normalized channel matrices. Random scatterer
environments are reused across compared cases within each Monte Carlo
realization.

## Automated Checks

```matlab
run_tests
```

The tests verify:

- DFT matrix unitarity.
- Ideal capacity invariance.
- Fixed-seed FAS-UAV determinism.
- Port-index and matrix-dimension validity.
- Separate raw-CIR and normalized-channel paths.
- Modeling-error equation behavior.
- Sparsity and operation-count metrics.
- Required output files and finite values.
